# Правила для ИИ-верстки (AI-assisted layout)

Краткие правила, которые нужно соблюдать при генерации HTML/CSS разметки для этого проекта.

## Общие принципы
- Используйте семантические теги HTML5: `header`, `main`, `section`, `article`, `nav`, `footer`.
- Не добавляйте inline-стили; все классы должны соответствовать существующей CSS-архитектуре.
- Отдавайте предпочтение переиспользуемым компонентам из `src/css/components/` и `src/css/sections/`.

## Именование классов
- Следуйте BEM: `block__element--modifier`.
- Не придумывайте глобальные селекторы — создавайте классы в формате компонента.

## CSS и переменные
- Используйте CSS-переменные из `src/css/base/root.css` для цветов, отступов и размеров.
- Не добавляйте новые глобальные переменные без согласования с командой.

## Адаптивность и изображения
- Верстка должна быть desktop-first; используйте существующие медиа-точки проекта.
- Для изображений используйте `picture`/`srcset` с AVIF/WEBP/PNG fallback и храните ассеты в `img/`.
- Минимизмруйте медиа запросы используя `clamp`

## Доступность (a11y)
- Все декоративные изображения — `alt=""`, значимые — с осмысленным `alt`.
- Используйте `aria-*` и фокусируемые элементы для интерактивных компонентов.

## JavaScript
- Не создавайте новых глобальных скриптов; инициализацию интегрируйте в `src/js/components/` и `src/js/main.js`.
- Для динамических компонентов используйте event delegation и проверяйте существование элементов.

## Где создавать файлы
- Новые страницы — в `src/pages/`.
- Новые CSS-файлы компонентов — в `src/css/components/` или `src/css/sections/`.

## Ограничения
- Не подключайте сторонние библиотеки без обсуждения.
- Не изменяйте `index.html` и глобальные шаблоны без PR.
